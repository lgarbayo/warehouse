# 📦 Instrucciones de Entrega

**Proyecto Warehouse - Sistemas Inteligentes 2025-2026**  
**Universidad de Vigo**

---

## 📋 Contenido a Entregar

### 1. Memoria del Proyecto (Obligatorio)

**Archivo:** `memoria.pdf`

Ubicación: Colocar directamente en esta carpeta `doc/`


>Se espera que la memoria tenga la calidad técnica y redacción adecuada, con diagramas claros y explicaciones detalladas. Deberá explicar las decisiones de diseño, la lógica de los agentes, la interacción entre ellos y cómo se han abordado los objetivos del proyecto. Además, debe incluir dificultades encontradas y cómo se han solucionado, así como referencias y posibles mejoras futuras si el tiempo lo permite.

> La memoria debe ser clara, concisa y bien estructurada, facilitando la comprensión del proyecto a cualquier lector.

> No se espera que sea un documento extenso, pero sí completo y bien estructurado.
---

## ✅ Checklist de Entrega

Antes de entregar, verificar:

### Código
- [ ] Todos los archivos `.asl` están implementados
- [ ] El sistema ejecuta correctamente (`jason warehouse.mas2j`)
- [ ] No hay errores de compilación
- [ ] Los robots realizan tareas correctamente
- [ ] El scheduler asigna tareas apropiadamente
- [ ] El supervisor monitoriza el sistema
- [ ] La gestión de errores funciona

### Memoria
- [ ] Archivo `memoria.pdf` en `doc/`
- [ ] Todas las secciones requeridas están completas
- [ ] Métricas de rendimiento documentadas
- [ ] Referencias bibliográficas (si aplica)
- [ ] Revisión ortográfica realizada

### Código Fuente
- [ ] Archivos `.asl` copiados a `doc/src/agt/`
- [ ] Código comentado apropiadamente
- [ ] Sin código de debug innecesario
- [ ] Formato consistente

### Empaquetado
- [ ] Archivo ZIP creado correctamente
- [ ] Nombre del archivo: `warehouse_grupoXX.zip`
- [ ] Tamaño razonable (< 50 MB)
- [ ] Carpetas `.git`, `.gradle`, `build` excluidas

### Otros
- [ ] Todos los miembros del grupo listados en la memoria
- [ ] Preparados para la defensa oral
- [ ] Entregado antes de la fecha límite

---

## ⚠️ Advertencias Importantes

### ❌ NO Hacer:
- **NO incluir** binarios compilados (carpeta `build/`)
- **NO incluir** `.git/` en el ZIP
- **NO entregar** fuera de plazo (penalización según normativa)
- **NO copiar** código de otros grupos (plagio = suspenso)

### ✅ SÍ Hacer:
- **SÍ probar** exhaustivamente antes de entregar
- **SÍ comentar** el código apropiadamente
- **SÍ incluir** todas las secciones en la memoria
- **SÍ revisar** el checklist completo
- **SÍ preparar** la defensa oral con todo el grupo

---

## 🎤 Defensa Oral

### Formato
- Duración: 15-20 minutos
- Presentación + preguntas
- Todos los miembros deben participar

### Contenido a Presentar
1. Demostración del sistema funcionando
2. Explicación de la arquitectura
3. Decisiones de implementación clave
4. Dificultades y soluciones
5. Métricas y resultados

### Preguntas Frecuentes

**P: ¿Puedo usar librerías externas?**

**P: ¿Cuántas páginas debe tener la memoria?**  
R: No hay mínimo/máximo. Debe cubrir todos los puntos requeridos con claridad.

**P: ¿Qué pasa si nuestro sistema tiene bugs?**  
R: Documentad los problemas conocidos en la memoria. Se valora la honestidad.
